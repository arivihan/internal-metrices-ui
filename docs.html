Proxy Configuration

Vite dev server proxies API requests to avoid CORS issues during development.

vite.config.ts Proxy Settings ▼
Local Path Target Server Rewrite Rule Purpose
/api/* https://platform-dev.example.com /api → /internal-metrics Main API requests
/analytics-api/* https://analytics.example.com /analytics-api → / Analytics & user activity tracking 
// vite.config.ts
import { defineConfig } from 'vite'

export default defineConfig({
  server: {
    port: 3000,
    proxy: {
      '/api': {
        target: 'https://platform-dev.example.com',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/api/, '/internal-metrics'),
      },
      '/analytics-api': {
        target: 'https://analytics.example.com',
        changeOrigin: true,
        rewrite: (path) => path.replace(/^\/analytics-api/, ''),
      },
    },
  },
})
💡 How Proxy Works

When you call fetch('/api/users'), Vite intercepts it and forwards to https://platform-dev.example.com/internal-metrics/users

State Management (Preact Signals)

💡 Why Signals?

Preact Signals provide lightweight, reactive state management without the boilerplate of Redux or Context API. Changes automatically trigger re-renders only in components that use the signal.

signals/auth.ts Authentication State ▼ 
// Core signals
accessToken: signal<string | null>     // JWT token
currentUser: signal<User | null>       // User object
userLoading: signal<boolean>           // Loading state

// Computed signals (auto-update)
isAuthenticated = computed(() => !!accessToken.value)
isAdmin = computed(() => currentUser.value?.role.includes('ADMIN'))
userRoles = computed(() => currentUser.value?.role ?? [])

// Functions
setAuth(token)      // Store token in signal + cookie
clearAuth()         // Logout - clear all auth state
fetchCurrentUser()  // Get user profile from API
signals/sidebar.ts Sidebar State ▼ 
sidebarData: signal<SidebarConfig | null>
sidebarLoading: signal<boolean>
sidebarError: signal<string | null>

// Functions
loadSidebarData()        // Fetch from API
refreshSidebarData()     // Force refresh
getFilteredDrawerItems() // Filter by role
signals/theme.ts Theme State ▼ 
theme: signal<'light' | 'dark' | 'system'>

// Functions
setTheme(theme)       // Set theme
toggleTheme()         // Cycle through themes
getResolvedTheme()    // Resolve 'system' to actual theme

// Effect: auto-applies theme to document.documentElement

Authentication

Auth Flow ▼ 
1. User enters phone number
   → POST /secure/auth/send-otp

2. User enters OTP
   → POST /secure/auth/verify-otp
   ← Returns { token, user }

3. Token stored
   → Cookie: avToken (domain: .arivihan.com, 7 days)
   → Signal: accessToken.value

4. All API requests include
   → Header: avToken: {token}
   → Credentials: include (for cookies)

5. On app load
   → useAuthInit() hook checks for token
   → Fetches user profile: GET /secure/auth/me
